# Remote Kiosk Device Management System

**Summary:** Web portal for live monitoring and remote control of donation kiosk fleets.

**Problem:** Supporting kiosks in the field meant blind debugging — no visibility into
battery, connectivity, or uptime, and every fix required someone on-site.

**Approach:** Built a web-based management portal integrating KLR REST APIs for device
insights (battery, Wi-Fi, uptime) and remote commands (reboot, screenshot), with live
status views for the support team.

**Impact:** Materially improved support response time — most issues now diagnosed and
fixed remotely.

**Tech stack:** .NET, REST APIs (KLR), SQL Server, JavaScript/Alpine.js
